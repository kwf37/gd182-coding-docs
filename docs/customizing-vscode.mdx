# Customizing the Code Editor

This page has a few handy settings and extensions that can make HTML/CSS coding easier.

## Format on Save

This is a must! Having files well-formatted makes them easier to read and modify. It can also alert you to
issues in your code because if you have an error, sometimes VSCode won't be able to reformat your code.

To set this up, open the VSCode settings either through the top left menus (`Code -> Settings -> Settings`),
or with the shortcut `CMD + ,` (or `CTRL + ,` on Windows).

Search for "format on save," and click the checkbox on the code to enable this setting:
![VSCode Format on Save Setting](/img/vscode-format-on-save.png)

After that, save, save, save!

## Picking Color Themes

We all love playing with colors! In the settings, search for "color," and you should see a dropdown like the following:

Try out some different options and see what you like!
![VSCode Color Theme Setting](/img/vscode-color-themes.png)

If the default ones aren't enough, you can find more at sites like https://vscodethemes.com/.

## Complete CSS Semicolon

This may be a default now, but you can configure VSCode to autocomplete CSS properties by adding the semicolon for you.
Search for "complete property" and you should find the following setting to check:
![VSCode Complete CSS Properties](/img/vscode-complete-css-semicolon.png)

## Auto Rename HTML Tags

This is a VSCode _extension_, rather than a setting. When you edit HTML tags, it will automatically rename the closing
tag to match. This is helpful if you realize you used the wrong tag somewhere and want to replace it quickly.

To install this, first look for an icon made of small blocks on the very left of the editor.
Clicking it should open a search bar where you can find extensions other programmers made to enhance VSCode.

Search for "Auto Rename Tag," and click the top one by Jun Han.

On the page that pops up, click `install`.

![VSCode Install Auto Rename Tag Extension](/img/vscode-auto-rename-tag-extension.png)
